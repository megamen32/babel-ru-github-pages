# Библиотека Вавилона · русская версия для GitHub Pages

Статический MVP русской «Библиотеки Вавилона».

Главный принцип:

```text
URL → seed → генератор → одна и та же страница
```

Сайт не использует backend, базу данных и сборку. Его можно положить прямо в GitHub Pages.

## Что уже есть

- постоянная ссылка на каждую страницу через `/#/room/.../wall/.../shelf/.../book/.../page/...`;
- русская кириллическая генерация текста;
- три режима текста:
  - кириллический хаос;
  - словесный шум;
  - псевдопророчество;
- поиск фразы;
- подсветка найденной фразы;
- ссылка на результат поиска, которой можно поделиться;
- случайная страница;
- переход на предыдущую и следующую страницу;
- ручной переход по адресу вида `зал/стена/полка/том/страница`;
- копирование ссылки;
- копирование текста страницы;
- скачивание страницы в `.txt`;
- избранное в `localStorage`;
- `.nojekyll` для GitHub Pages;
- workflow для деплоя через GitHub Actions.

## Быстрый запуск локально

Можно просто открыть `index.html` в браузере.

Или запустить локальный сервер:

```bash
python3 -m http.server 8080
```

После этого открыть:

```text
http://localhost:8080
```

## Деплой на GitHub Pages

```bash
git init
git add .
git commit -m "Initial Russian Library of Babel"
git branch -M main
git remote add origin git@github.com:USERNAME/babel-ru.git
git push -u origin main
```

Потом в репозитории GitHub:

```text
Settings → Pages → Build and deployment → Source → GitHub Actions
```

Workflow `.github/workflows/pages.yml` сам опубликует сайт.

## Почему используется `/#/`

GitHub Pages — статический хостинг. Hash-router безопаснее для проекта без backend: браузер открывает один и тот же `index.html`, а приложение уже внутри страницы читает адрес после `#`.

Пример постоянной ссылки:

```text
https://USERNAME.github.io/babel-ru/#/room/17/wall/2/shelf/4/book/119/page/36?mode=chaos
```

## Важное ограничение поиска

В MVP поиск работает так: фраза кодируется в URL результата и детерминированно вставляется в найденную страницу. Это честная статическая имитация борхесовской идеи без хранения бесконечной базы страниц.

Если нужна более строгая математическая версия, следующий шаг — сделать обратимое кодирование текста в координаты страницы, чтобы координаты сами содержали фразу без параметра `q`.
