# Русская Библиотека Вавилона — реальный обратимый алгоритм

Эта версия не хранит страницы, не строит индекс и не кладёт фразу отдельным payload.

Алгоритм:

```text
фиксированный алфавит
+
фиксированная длина страницы
=
каждая страница является числом в системе счисления по основанию alphabet.length
```

Открытие страницы:

```text
base36-адрес → BigInt → текст страницы
```

Создание страницы с фразой:

```text
фраза + номер варианта
→ строится полная страница фиксированной длины
→ вся страница кодируется в BigInt
→ BigInt кодируется в base36-адрес
```

Ссылка содержит не фразу отдельно, а число всей страницы.

## Почему страниц много

Если алфавит содержит `N` символов, а страница имеет длину `L`, то количество страниц:

```text
N^L
```

В текущей версии:

```text
alphabet.length ^ PAGE_LENGTH
```

Параметры находятся в `app.js`:

```js
const ALPHABET = "...";
const PAGE_LENGTH = 900;
```

## Почему нельзя показать все страницы с фразой

Страниц с данной фразой тоже астрономически много. Поэтому интерфейс показывает несколько вариантов:

```text
Вариант 1
Вариант 2
Вариант 3
...
```

Каждый вариант — настоящая страница из пространства всех возможных страниц.

## Формат ссылки

```text
#/v1/page/<base36-address>
```

## Важное правило

Не менять алгоритм `v1` после публикации. Если нужно менять алфавит, длину страницы или кодирование, нужно добавлять `v2`, чтобы старые ссылки не ломались.

## Локальный запуск

```bash
python3 -m http.server 8000
```

Потом открыть:

```text
http://localhost:8000/
```
