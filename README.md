# Русская Библиотека Вавилона — координатная версия

Это версия без хранения страниц, без индекса и без payload с фразой.

Главное улучшение v3: координаты теперь не декоративные.  
Они реально получаются из числа страницы делением с остатком.

## Принцип

```text
страница → BigInt → сектор / зал / стена / полка / том / лист
сектор / зал / стена / полка / том / лист → BigInt → страница
```

## Что есть

- настоящая координатная система;
- проверка диапазона адреса;
- соседние страницы;
- предыдущий/следующий том;
- shareable-страница результатов `#/find?...`;
- пагинация вариантов через `offset`;
- классический режим `v1c`: 3200 символов, 40 строк × 80;
- стандартный режим `v1`: 900 символов;
- короткий режим `v1s`: 300 символов;
- режим “кодировать текст в страницу”;
- проверка обратимости с подробными шагами;
- избранное через `localStorage`;
- скачивание текста;
- скачивание PNG-карточки;
- GitHub Pages workflow.

## Формат координатной ссылки

```text
#/v1/sector/1/hall/1/wall/1/shelf/1/volume/1/page/1
```

## Raw-адрес тоже поддерживается

```text
#/v1/a/abc12345-def67890
```

Но при открытии raw-адреса страница показывает настоящие координаты.

## Поиск фраз

Страница результатов:

```text
#/find?version=v1&q=текст&offset=0&count=8&strategy=random
```

Сами страницы библиотеки не содержат фразу в URL.  
Они содержат координаты страницы.  
Подсветка хранит только диапазон:

```text
?hl=143:37
```

## Важное правило

Нельзя менять уже опубликованную версию алгоритма:

- алфавит;
- длину страницы;
- lineWidth;
- pagesPerVolume;
- volumesPerShelf;
- shelvesPerWall;
- wallsPerHall;
- hallsPerSector;
- порядок кодирования.

Для новой логики добавлять `v2`.

## Локальный запуск

```bash
python3 -m http.server 8000
```

Открыть:

```text
http://localhost:8000/
```

## GitHub Pages

```text
.github/workflows/pages.yml
```

В GitHub включить:

```text
Settings → Pages → Build and deployment → Source → GitHub Actions
```
