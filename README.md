# Русская Библиотека Вавилона — детерминированная версия для GitHub Pages

Эта версия не хранит страницы и не строит индекс.

Главное правило:

```text
URL → seed → тот же самый текст
```

То есть страница однозначно восстанавливается из ссылки.

## Что есть

- GitHub Pages-ready SPA;
- hash-router, безопасный для статического хостинга;
- постоянные ссылки на страницы;
- генерация страницы только из координат;
- версия алгоритма в URL: `v1`;
- режим фразовой страницы: фраза кодируется в ссылке и поэтому тоже восстанавливается без сервера;
- копирование ссылки;
- копирование текста;
- скачивание страницы `.txt`;
- GitHub Actions workflow для Pages.

## Формат обычной страницы

```text
#/v1/room/17/wall/2/shelf/4/book/119/page/36
```

## Формат фразовой страницы

```text
#/v1/phrase/<payload>
```

Payload содержит фразу и адрес. Это важно: без хранения и индекса нельзя честно восстановить произвольную фразу из одного хеша.

## Локальный запуск

```bash
python3 -m http.server 8000
```

Открыть:

```text
http://localhost:8000/
```

## GitHub Pages

В репозитории уже есть workflow:

```text
.github/workflows/pages.yml
```

В GitHub включить:

```text
Settings → Pages → Build and deployment → Source → GitHub Actions
```

## Важное правило разработки

Не менять генератор `v1` после публикации.

Если нужен другой стиль генерации, добавить новый маршрут и новый алгоритм:

```text
#/v2/...
```

Так старые ссылки не сломаются.
