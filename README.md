# Вавилон v8.0 — Гексагональная Бесконечность

> Вселенная, которую другие называют Библиотекой, состоит из огромного, возможно, бесконечного числа шестигранных залов.
> — Хорхе Луис Борхес, «Вавилонская библиотека»

Не генератор. Не ИИ. Не база данных.

Математически адресуемая библиотека, где каждая возможная страница из 4096 символов уже существует и вычисляется из координат за ноль секунд. Ни один байт текста не хранится на сервере — всё определяется обратимым преобразованием над 32768-битными числами.

## Что это

Вавилон — русскоязычная реализация идеи Library of Babel, переписанная с нуля под современный веб. Каждая страница:

- фиксированная длина — 4096 символов (один Telegram-пост);
- алфавит — 256 символов = 1 байт (кириллица, латиница, цифры, знаки, 149 эмодзи);
- существует всегда — вычисляется, не хранится;
- обратима — по тексту можно найти координаты, по координатам — текст;
- перенос строки — полноценный символ алфавита, форма текста сохраняется.

## Координаты: X, Y, Z — бесконечная полка

Вместо шестиуровневой иерархии Борхеса (сектор → зал → стена → полка → том → страница) используется три координаты:

- **X** — позиция зала на бесконечной 2D-карте;
- **Y** — позиция зала на бесконечной 2D-карте;
- **Z** — номер страницы в этом зале (BigInt, 1 и выше).

Борхесовский формат (стена/полка/том/лист) сохранён как display-only — для отображения в заголовках. Внутренняя архитектура работает только с X, Y, Z.

URL формат: `#/x/{x}/y/{y}/z/{z}`

Старые URL с `/w/.../sh/.../v/.../p/...` автоматически редиректятся на новый формат.

## Префиксный декодер — гравитация языка

Ключевое архитектурное нововведение v8.0: страница **декодируется** из адреса, а не генерируется из PRNG.

### Как работает

```
(x, y, z) → координатная перестановка → внутренний адрес (BigInt)
         → поток битов → префиксный декодер Level1 (тип токена)
         → префиксный декодер Level2 (конкретный токен)
         → текст страницы 4096 символов
```

### Канонический Хаффман

10 типов токенов (русские слова, английские слова, русские фразы, английские фразы, пунктуация, точка, пробел, перенос строки, эмодзи, raw-символ) кодируются префиксными кодами разной длины. Частые токены получают короткие коды → малые адреса → страницы с человеческим текстом группируются в начале координат.

Это и есть **«гравитация языка»**: чем естественнее текст, тем короче его адрес.

### Конечный автомат

8 состояний (START, AFTER_RU, AFTER_EN, AFTER_SPACE, AFTER_DOT, AFTER_PUNCT, AFTER_NL, AFTER_EMOJI) управляют грамматикой генерируемого текста. После русского слова вероятнее пробел или пунктуация, после точки — пробел, после эмодзи — пробел или новое слово.

### Температура

Z-координата определяет «температуру» страницы — степень удалённости от человеческого текста:

| Z | Температура | Результат |
|---|---|---|
| 1–100 | ~0.03 | Читаемый русский/английский текст |
| 10^3 | ~0.30 | Разговорный, с вкраплениями шума |
| 10^6 | ~0.60 | Смешанный: слова + случайные символы |
| 10^10 | ~1.00 | Полный хаос |

### Обратимый поиск

Поиск работает через **обратное кодирование**: фраза → токены → префиксные коды → поток битов → адрес (BigInt) → координаты. Найденный адрес — это **честное место** в библиотеке, где фраза действительно существует. Не вставка в случайную страницу, а реальное декодирование.

## Аффинная перестановка (legacy-слой)

Для совместимости со старыми base64-адресами сохранена аффинная перестановка над Z/(2^32768):

```
contentNumber = (bookIndex × C + OFFSET)  mod 2^32768
bookIndex     = (contentNumber − OFFSET) × I  mod 2^32768
```

- **C** — нечётная константа, построенная мозаикой 64-битного паттерна по всей ширине 32768 бит;
- **I** — мультипликативно обратный элемент (лемма Гензеля);
- **OFFSET** — высокоэнтропийная константа.

Avalanche effect: соседние координаты дают ~0% сходства страниц.

## Алфавит: 256 = 2^8

| Диапазон | Количество | Содержание |
|----------|-----------|------------|
| 0 | 1 | Пробел |
| 1 | 1 | Перенос строки `\n` |
| 2–34 | 33 | Кириллица а–я + ё |
| 35–60 | 26 | Латиница a–z |
| 61–70 | 10 | Цифры 0–9 |
| 71–106 | 36 | Знаки препинания |
| 107–255 | 149 | Эмодзи |

Визуально совпадающие буквы (А/А, Е/Е, К/К и т.д.) хранятся как отдельные записи. Поиск автоматически отображает их в русские аналоги.

### Модульная архитектура

Код разбит на логические модули, каждый в своём IIFE. Модули общаются через `app.library` — общее пространство имён. Порядок загрузки (через `defer`):

1. `lib-prefix-codec.js` → `app.library._prefix` — Хаффман + битовые потоки
2. `lib-token-table.js` → `app.library._tokenTable` — словарь + состояния
3. `lib-address-codec.js` → `app.library._addressCodec` — декодер/энкодер адресов
4. `lib-coordinate-permutation.js` → `app.library._coordPerm` — перестановка координат
5. `lib-tokens.js` → `app.library._tokens` — PRNG-декодер (fallback)
6. `lib-core.js` → `app.library._core` — аффинная перестановка, X/Y/Z
7. `lib-fillers.js` → `app.library._fillers` — генерация заполнений
8. `lib-classifier.js` → `app.library._classifier` — классификация страниц
9. `lib-api.js` → `app.library` — публичный API, затем cleanup временных пространств имён

### Web Worker

BigInt-операции над 32768-битными числами блокируют главный поток на мобильных. Весь тяжёлый вычислительный код вынесен в `worker.js`:

- Поиск текста (префиксное кодирование фразу → адрес)
- Декодирование страницы (адрес → текст)
- Пакетная загрузка корешков
- Координатные преобразования

### Шутки и цитаты

Во время поиска Библиотекарь травит анекдоты и мудрые цитаты. Полностью офлайн — детерминированная генерация из поискового запроса:

- **Шутки** — комбинаторные анекдоты из заготовленных opener/subject/twist блоков;
- **Наблюдения** — библиотечная мудрость с подстроечным упоминанием фразы;
- **Цитаты** — философские наблюдения о поиске в бесконечности;
- **Библиотекарь** — персональное послание пока идут вычисления.

Никаких сетевых запросов. Работает и без интернета.

### PWA

Service Worker + manifest.json → установка на телефон как приложение, offline-first, иконка на рабочем столе.

## Почему это выглядит так

Большинство реализаций «Библиотеки Борхеса» — это скучная сетка со случайным текстом, или криптографическая демка без души.

Вавилон — другое дело. Библиотека должна ощущаться как **место**. Как пространство, где хочется блуждать. Где можно включить тему «Мессенджер» и читать страницы как чат от призрачного Библиотекаря, который подшучивает пока ищет. Или переключиться на «Книжная полка» и смотреть на корешки книг — некоторые с настоящим текстом, некоторые с шумом, некоторые пустые.

Каждая тема — не просто скин, а **способ взаимодействия** с бесконечностью.

## Борхес

Проект вдохновлён рассказом «Вавилонская библиотека» (La biblioteca de Babel, 1941).

Борхес описал библиотеку из шестигранных залов, где каждый зал содержит определённое количество книг с определённым количеством страниц, и эти книги содержат все возможные комбинации символов. Значит — каждую возможную книгу. Включая твою биографию, твою смерть, и рецепт борща.

Вавилон делает этот мысленный эксперимент **исполнимым**: каждый текст реально существует по вычислимому адресу, и к нему можно перейти.

## Ключевые решения

### Почему префиксный декодер, а не PRNG?

PRNG-генератор даёт «примерно похожий на текст» результат, но связь между адресом и содержимым — случайная. Префиксный декодер делает связь **честной**: малые адреса → частые токены → человекоподобный текст. Это создаёт «гравитацию языка» к началу координат и делает поиск обратимым.

### Почему 256, а не 29?

29-символьный алфавит требует логарифмов и деления с остатком. 256 = чистые побитовые сдвиги (`<<8`, `&255`). Быстрее, проще, идеально для BigInt.

### Почему 4096?

4096 = максимальная длина сообщения в Telegram. Библиотека XXI века работает с постами, а не с бумажными страницами.

### Почему X,Y,Z, а не 6 уровней?

Шестиуровневая иерархия Борхеса (сектор/зал/стена/полка/том/лист) — красиво для описания, но неудобна для URL и навигации. Три координаты X,Y,Z покрывают всё пространство и дают простую прогрессию: `#/x/0/y/0/z/1`, `#/x/0/y/0/z/2`, ... Борхесовский формат сохранён для отображения.

### Почему аффинная перестановка, а не хеш?

Перестановка обратима — по содержимому можно восстановить координаты. Хеш однонаправлен. Обратимость критична для поиска.

### Почему полный офлайн?

Каждый сетевой запрос — точка отказа. rzhunemogu.ru падает, forismatic.com не отвечает, CORS-прокси закрываются. Словарь 10 000 русских слов встроен в `words.js`, шутки генерируются детерминированно из запроса, Service Worker кэширует приложение. Вавилон работает и в метро, и в самолёте.
