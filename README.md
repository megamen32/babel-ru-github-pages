# Вавилон v8.0 — Гексагональная Бесконечность

> Вселенная, которую другие называют Библиотекой, состоит из огромного, возможно, бесконечного числа шестигранных залов.
> — Хорхе Луис Борхес, «Вавилонская библиотека»

Не генератор. Не ИИ. Не база данных.

Математически адресуемая библиотека, где каждая возможная страница из 4096 символов уже существует и вычисляется из координат за ноль секунд. Ни один байт текста не хранится на сервере — всё определяется обратимым преобразованием над 32768-битными числами.

---

## Что это

Вавилон — русскоязычная реализация идеи Library of Babel, переписанная с нуля под современный веб. Каждая страница:

- фиксированная длина — 4096 символов (один Telegram-пост);
- алфавит — 256 символов = 1 байт (кириллица, латиница, цифры, знаки, 149 эмодзи);
- существует всегда — вычисляется, не хранится;
- обратима — по тексту можно найти координаты, по координатам — текст;
- перенос строки — полноценный символ алфавита, форма текста сохраняется.

## Как это работает

### Алфавит: 256 = 2⁸

Каждый символ кодируется ровно 1 байтом. Полная таблица:

| Диапазон | Количество | Содержание |
|----------|-----------|------------|
| 0 | 1 | Пробел |
| 1 | 1 | Перенос строки `\n` |
| 2–34 | 33 | Кириллица а–я + ё |
| 35–60 | 26 | Латиница a–z |
| 61–70 | 10 | Цифры 0–9 |
| 71–106 | 36 | Знаки препинания |
| 107–255 | 149 | Эмодзи |

Визуально совпадающие буквы (А/А, Е/Е, К/К и т.д.) хранятся как отдельные записи — для них хватает места.

### Страница = число

4096 символов × 8 бит = 32768 бит. Каждая страница — это 32768-битное число. Всё пространство библиотеки: 2³²⁷⁶⁸ возможных страниц.

### Аффинная перестановка

Чтобы соседние координаты не давали похожий текст, применяется аффинный шифр над Z/(2³²⁷⁶⁸):

```
contentNumber = (bookIndex × C + OFFSET)  mod 2³²⁷⁶⁸
bookIndex     = (contentNumber − OFFSET) × I  mod 2³²⁷⁶⁸
```

- **C** — нечётная константа, построенная мозаикой 64-битного паттерна по всей ширине 32768 бит (чередование с инверсией для высокого Hamming distance);
- **I** — мультипликативно обратный элемент (лемма Гензеля для 2-адических чисел);
- **OFFSET** — высокоэнтропийная константа, решает проблему нулевой страницы (0 × C = 0 без смещения).

**Критический фикс avalanche**: ранний прототип имел PERM_C ≈ 2⁶³ при модуляторе 2³²⁷⁶⁸ — соседние страницы совпадали на 99.8%. После мозаичной константы C соседние страницы имеют ~0% сходства.

### Геометрия библиотеки

Навигация по иерархии Борхеса:

```
зал → стена → полка → том → страница
```

- 410 страниц в томе
- 32 тома на полке
- 5 полок на стене
- 4 стены в зале
- 20 залов в секторе
- Секторы нумеруются от 1 до бесконечности

### Координаты XY

Двумерная навигация по залам через парирование Шудзика (Szudzik pairing function) — биективное отображение Z² → Z. Каждый зал имеет координаты (X, Y), которые однозначно определяют сектор и номер зала.

### Поиск

Любой введённый текст можно найти в библиотеке. Алгоритм:

1. Нормализация текста (lowercase, collapse whitespace);
2. Токенизация в индексы алфавита;
3. Выбор режима заполнения (пустота / шум / слова);
4. Вставка фразы в случайную позицию страницы;
5. Обратная перестановка → координаты страницы.

Результат: до 18 вариантов расположения фразы на разных страницах.

## Архитектура

Чистый статический сайт. Ноль серверной логики. Всё работает в браузере.

```
babel-ru-github-pages/
├── index.html              # SPA-shell
├── 404.html                # SPA routing redirect
├── css/
│   └── style.css           # 5 тем через CSS custom properties
├── js/
│   ├── config.js           # Алфавит, константы, lazy-загрузка словаря
│   ├── utils.js            # Токенизация, рендеринг, утилиты
│   ├── library.js          # Ядро: перестановка, координаты, поиск
│   ├── worker.js           # Web Worker — асинхронный BigInt-движок
│   ├── worker-bridge.js    # Promise API + шутки/цитаты + typing indicator
│   ├── themes.js           # 5 рендереров + общие компоненты
│   ├── app.js              # Роутер, навигация, привязка событий
│   ├── hexweb.js           # Three.js 3D-визуализация паутины залов
│   └── storage.js          # localStorage: избранное, история
└── tests/
    └── test_adjacent_similarity.js
```

### Web Worker

BigInt-операции над 32768-битными числами блокируют главный поток на мобильных. Весь тяжёлый вычислительный код вынесен в `worker.js`:

- Поиск текста (создание вариантов)
- Декодирование страницы (индексы → текст)
- Пакетная загрузка корешков
- Координатные преобразования

### Удалённый словарь

Словарь русских слов (10 000) загружается по сети при первом поиске в режиме «Слова». Не捆绑ывается локально — 40 слов-заглушек для offline.

### Шутки и цитаты

Во время поиска Библиотекарь травит анекдоты и мудрые цитаты:

- **rzhunemogu.ru** — 18 категорий (анекдоты, афоризмы, стишки, программистские и т.д.) через CORS-прокси;
- **forismatic.com** — мудрые цитаты на русском (JSON API, CORS-friendly);
- **Фоллбэк** — библиотечные шутки от Библиотекаря.

## Почему это выглядит так

Большинство реализаций «Библиотеки Борхеса» — это скучная сетка со случайным текстом, или криптографическая демка без души.

Вавилон — другое дело. Библиотека должна ощущаться как **место**. Как пространство, где хочется блуждать. Где можно включить тему «Мессенджер» и читать страницы как чат от призрачного Библиотекаря, который подшучивает пока ищет. Или переключиться на «Книжную полку» и смотреть на корешки книг — некоторые с настоящим текстом, некоторые с шумом, некоторые пустые.

Каждая тема — не просто скин, а **способ взаимодействия** с бесконечностью.

## Борхес

Проект вдохновлён рассказом «Вавилонская библиотека» (La biblioteca de Babel, 1941).

Борхес описал библиотеку из шестигранных залов, где каждый зал содержит определённое количество книг с определённым количеством страниц, и эти книги содержат все возможные комбинации символов. Значит — каждую возможную книгу. Включая твою биографию, твою смерть, и рецепт борща.

Вавилон делает этот мысленный эксперимент **исполнимым**: каждый текст реально существует по вычислимому адресу, и к нему можно перейти.

## Технические детали

- **BigInt** — все вычисления над 32768-битными числами через native BigInt (Chrome 67+, Firefox 68+, Safari 14+);
- **Web Worker** — тяжёлые операции вынесены из главного потока;
- **SPA** — hash-роутинг, GitHub Pages 404.html redirect;
- **Координатные URL** — `#/page/s/1/h/3/w/2/sh/5/v/32/p/410` — стабильные, без хэшей;
- **Zero dependencies** — ни одного npm-пакета в продакшене;
- **5193 строки** кода (JS + CSS + HTML).
